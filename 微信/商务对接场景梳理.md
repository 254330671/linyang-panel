# 商务对接场景梳理

> 适用：聚合信息面板 App 未来可能遇到的商务合作
> 整理：2026-03-28

---

## 一、支付渠道对接

| 对接方 | 场景 | 你需要准备 | 对方关注 |
|--------|------|-----------|---------|
| **微信支付** | App 内订阅/购买 | 营业执照、法人身份证、银行卡 | 商户资质合规、交易流水 |
| **支付宝** | 同上 | 同上 | 同上 |
| **苹果 IAP** | iOS 版订阅 | Apple Developer 账号（¥688/年） | 审核合规、不绕过 IAP |
| **Stripe** | 海外用户支付 | 海外银行账户、公司/个人注册 | 合规、退款率 |

**可能的情况：**
- 微信/支付宝审核被驳回（经营范围不符、资料不全）
- 苹果审核卡壳（虚拟商品没走 IAP、功能描述不清）
- 费率谈判（流水大了可以申请更低费率）

---

## 二、应用商店上架

| 平台 | 要求 | 对接要点 |
|------|------|---------|
| **苹果 App Store** | Apple Developer 账号、软著、隐私政策 | 审核严格，反复被打回是常态 |
| **华为应用市场** | 软著、ICP备案、隐私合规检测 | 国内最大渠道，必须上 |
| **小米应用商店** | 软著、ICP备案 | 小米用户多 |
| **OPPO/vivo** | 软著、ICP备案 | 需分别申请开发者账号 |
| **应用宝（腾讯）** | 软著、ICP备案 | 微信生态关联 |

**可能的情况：**
- 每个商店审核标准不一，可能被要求改隐私政策、加权限说明
- 华为要求做"隐私合规检测"（有专门的检测工具）
- 苹果可能要求你解释"为什么需要这些权限"

---

## 三、API/云服务供应商

| 对接方 | 场景 | 合作方式 |
|--------|------|---------|
| **阿里云** | 服务器、数据库、OSS存储 | 云市场合作伙伴计划（返佣） |
| **腾讯云** | 备选云服务商 | 同上 |
| **OpenAI/DeepSeek** | AI 解析能力 | API 按量付费 |
| **高德地图** | 导航/地点搜索 | 免费额度 + 超出按量计费 |
| **极光推送/个推** | 消息推送 | 免费额度 + 付费套餐 |

**可能的情况：**
- 云厂商找你谈合作（你用户量起来了，他们想拉你上他们的云）
- API 供应商给折扣换你标注"Powered by XXX"
- 高德/百度地图商务找你谈企业版（如果你用他们 API 量大）

---

## 四、智能硬件合作

App 有跌倒检测功能，涉及手表/手环：

| 对接方 | 场景 | 合作方式 |
|--------|------|---------|
| **小米手环/手表** | 健康数据接入 | 小米 IoT 开放平台 API |
| **华为运动健康** | 同上 | 华为 Health Kit |
| **Apple Health** | iOS 健康数据 | HealthKit 框架 |
| **OPPO/vivo 健康** | 同上 | 各自开放平台 |

**可能的情况：**
- 硬件厂商主动找你（你的 App 带动他们设备销量）
- 你找他们谈数据接口开放（需要商务邮件/会议）
- 联合推广（"买 XX 手表送聚合面板 Pro 会员一年"）
- 厂商要求你在 App 里推荐他们的设备

---

## 五、企业客户/B端

| 场景 | 对接方式 | 注意事项 |
|------|---------|---------|
| **企业团购** | 企业批量购买 Pro 会员 | 需要开具发票（个体户可以开普票） |
| **私有化部署** | 企业想要在自己服务器上跑 | 需要单独报价，技术对接 |
| **API 输出** | 企业把你的 AI 解析能力嵌入他们系统 | 按调用量收费 |
| **定制开发** | 企业要加特定功能 | 按项目报价 |

**可能的情况：**
- 有公司想给员工买协作待办（团购 50-500 个账号）
- 学校/培训机构想用（需要教育优惠方案）
- 你拒绝不了的大客户要定制（要考虑值不值得接）

---

## 六、投资/融资

| 阶段 | 可能的对接方 | 你需要准备 |
|------|-------------|-----------|
| **种子轮** | 天使投资人、孵化器 | 商业计划书、产品 demo、用户数据 |
| **Pre-A** | 早期 VC | 用户增长曲线、留存率、付费转化率 |
| **A轮** | 机构 VC | 营收数据、团队、市场规模分析 |

**可能的情况：**
- 用户量到几万时开始有人关注
- 投资人问"你跟 XX 产品有什么区别"
- 要你做财务预测（收入模型、成本结构）
- 要你注册公司（个体户不够，要升级有限责任公司）

---

## 七、渠道/推广合作

| 对接方 | 场景 | 合作方式 |
|--------|------|---------|
| **应用商店推荐** | 争取首页推荐位 | 产品好 + 运营关系 |
| **KOL/博主** | 推广 App | 给免费会员 / 佣金分成 |
| **社群/论坛** | 种子用户获取 | 在知乎/小红书/即刻发帖 |
| **异业合作** | 跟互补产品互相导流 | "用 XX 记账 App 送聚合面板会员" |

**可能的情况：**
- 博主找你要评测用的免费账号
- 你主动找 KOL 谈推广（报价/分成模式）
- 应用商店运营找你要素材做专题

---

## 八、数据/隐私合规

| 对接方 | 场景 | 你需要准备 |
|--------|------|-----------|
| **网信办** | App 合规检查 | 隐私政策、数据安全评估 |
| **工信部** | ICP 备案抽查 | 备案信息真实有效 |
| **应用商店** | 隐私合规更新 | 每次更新可能被要求补充说明 |

**可能的情况：**
- 突然收到通知说你 App 不合规，要限期整改
- 用户投诉数据泄露，需要配合调查
- 新法规出台（如 AI 生成内容标注），需要改产品

---

## 九、税务相关

| 场景 | 说明 |
|------|------|
| **开发票** | 企业客户要求开增值税发票（个体户开普票，一般纳税人开专票） |
| **报税** | 季度销售额 ≤30 万免增值税，但要按时申报 |
| **个税** | 经营所得按 5-35% 五级累进税率 |

**可能的情况：**
- 企业客户说"我们只能对公转账，要发票"
- 流水大了税务局关注（个体户年收入 120 万以上要重点关注）
- 想省税 → 合理利用小微企业优惠政策

---

## 总结：按阶段梳理

| 阶段 | 必须对接 | 可能遇到 |
|------|---------|---------|
| **0-100 用户** | 微信/支付宝支付、应用商店 | 审核被拒、改隐私政策 |
| **100-1000 用户** | 推送服务、云服务优化 | 服务器成本上升、API 费用 |
| **1000-1万用户** | 硬件厂商（手表）、KOL推广 | 商务邮件变多、有人想合作 |
| **1万+用户** | 企业客户、投资人 | 需要注册公司、开票、融资谈判 |

---

*先心里有数，等遇到了再逐个处理。现在最重要的是先把产品做出来、上架、拿到第一批用户。*
