# 技能系统设计

> 来源：2026-03-28 对话
> 状态：讨论中，待确认

---

## 一、用户技能树

| 功能 | 说明 |
|------|------|
| **技能清单** | 用户自己标记会什么（做饭/骑车/编程/吉他等） |
| **想学清单** | 用户标记想学什么 |
| **技能进度** | 每个技能有进度条（入门→熟练→精通） |
| **学习记录** | 每次练习记一笔，自动算累计时长 |
| **里程碑** | "第一次独立做一顿饭""跑完5公里" |

## 二、与共创区联动

| 场景 | 触发 |
|------|------|
| 用户18岁 | 推荐"考驾照"，显示共创区经验（328人讨论过） |
| 用户标记"想学做饭" | 推荐共创区"新手菜谱"高赞帖 |
| 用户技能到"熟练" | 推荐进阶内容（从共创区拉） |
| 用户所在城市 | 推荐本地线下课程/活动 |

## 三、技能发现机制

AI 主动推荐，不光靠用户自己选：

- 看待办 → "你经常安排运动，要不要标记'跑步'技能？"
- 看年龄 → "这个阶段很多人开始学理财"
- 看共创区 → "最近很多人讨论吉他，感兴趣吗？"

## 四、技能与各模块连接

```
技能 → 待办：自动生成学习计划
技能 → 习惯：练习频次变成习惯追踪
技能 → 共创：学什么先看别人经验
技能 → 成长：技能进步 = 成长里程碑
技能 → 职业：关联职业发展需要的技能
```

## 五、数据结构

```
skills:
  ├── id, user_id
  ├── name（技能名称）
  ├── category（运动/生活/学习/职业/创意）
  ├── level（想学/入门/熟练/精通）
  ├── progress（0-100%）
  ├── total_hours（累计练习时长）
  ├── milestones[]（里程碑记录）
  ├── related_topics[]（关联共创话题）
  └── created_at, updated_at

skill_logs:
  ├── skill_id, date
  ├── duration_minutes
  ├── note（"今天练了和弦转换"）
  └── mood（练习心情）
```

## 六、商业价值

技能功能天然跟共创区绑定，形成正向飞轮：

```
用户学技能
  → 看共创区经验
  → 共创区更活跃
  → 推荐库更丰富
  → 吸引更多用户
  → 更多技能经验沉淀
```
