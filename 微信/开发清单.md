# 聚合信息面板 — 网页版开发清单

> Phase 1 核心目标：智能待办 + 动作链接 + 浏览器通知
> 纯前端，localStorage 持久化，调用户自己的 API

---

## 一、基础框架

- [ ] 确认现有 index.html 结构，划分功能区
- [ ] 待办数据结构设计（id/text/time/owner/visibility/status/source/actionLinks）
- [ ] localStorage 读写封装（统一 save/load 接口，为后续加密预留）
- [ ] Tab/分区导航逻辑整理（天气/待办/健康/成长/共创等）

## 二、智能待办 — AI 解析

- [ ] 用户输入框（支持一句话输入）
- [ ] API 调用封装（OpenAI 兼容格式，用户自填 API Key + Endpoint）
- [ ] Prompt 设计：一句话 → 结构化 JSON（文本/时间/分类/优先级）
- [ ] 解析结果预览 → 用户确认或修改 → 存入待办列表
- [ ] 解析失败降级：直接存为纯文本待办

## 三、动作链接

- [ ] 导航：解析地点 → 生成高德地图 URI（`https://uri.amap.com/navigation?...`）
- [ ] 购物：解析商品 → 生成淘宝/京东搜索链接
- [ ] 通讯：识别手机号/邮箱 → `tel:` / `mailto:` 链接
- [ ] 学习：识别课程/考试 → 搜索链接
- [ ] 健康：识别就医 → 提醒设置
- [ ] 动作链接在待办卡片上以按钮展示

## 四、待办管理

- [ ] 待办列表展示（按时间排序，无时间的放底部）
- [ ] 待办状态切换（待办/已完成/已取消）
- [ ] 待办编辑（修改文本/时间/分类）
- [ ] 待办删除
- [ ] 已完成待办归档（折叠或单独 Tab）
- [ ] 按分类筛选（全部/今日/本周/购物/导航/通讯等）

## 五、浏览器通知提醒

- [ ] 请求 Notification 权限（首次使用时弹窗说明）
- [ ] 有时间的待办 → 到点触发浏览器通知
- [ ] 通知点击 → 跳转到对应待办
- [ ] 22:30 汇总提醒（今日未完成待办）
- [ ] "推迟到明天"一键操作
- [ ] 通知开关（用户可关闭）

## 六、速记

- [ ] 快速输入框（独立于待办，纯文本记录）
- [ ] 速记列表（按时间倒序）
- [ ] 速记可一键转为待办
- [ ] 速记删除

## 七、番茄钟

- [ ] 25 分钟倒计时
- [ ] 开始/暂停/重置
- [ ] 完成后通知
- [ ] 今日完成次数统计
- [ ] 可关联当前待办

## 八、数据导出/导入

- [ ] 导出全部数据为 JSON 文件
- [ ] 导入 JSON 恢复数据（为未来迁移到 App 准备）
- [ ] 导出文件名带日期（`panel-backup-2026-03-28.json`）

## 九、UI/体验

- [ ] 响应式布局（桌面侧栏 / 手机底部导航）
- [ ] 深色/浅色主题切换
- [ ] PWA manifest + service worker（离线可用）
- [ ] 加载状态/空状态提示
- [ ] 操作反馈（toast 提示）

## 十、测试清单

- [ ] API 调用：输入一句话，检查解析结果是否准确
- [ ] 动作链接：点击后是否正确跳转
- [ ] 通知：设置 1 分钟后的待办，检查是否弹出
- [ ] 数据持久化：刷新页面数据不丢失
- [ ] 导出/导入：导出后导入，数据一致
- [ ] 手机端：在手机浏览器打开，检查布局和交互
- [ ] PWA：添加到主屏幕，离线打开

---

## 开发顺序建议

1. **先跑通 AI 解析**（核心功能，其他都依赖它）
2. **再做动作链接**（解析结果的直接输出）
3. **待办管理基础操作**（列表/状态/编辑/删除）
4. **通知提醒**（有时间待办才有意义）
5. **速记 + 番茄钟**（独立功能，不阻塞主线）
6. **导出/导入**（最后做，但很重要）
7. **UI 打磨**（贯穿始终，但优先级最低）
