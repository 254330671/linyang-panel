# APP 收费方式大全

> 整理：2026-03-28
> 用途：为聚合信息面板 App 选择合适的收费模式做参考

---

## 一、一次性买断

用户付一次钱，永久使用。

| 项目 | 说明 |
|------|------|
| **例子** | Notability 旧版、买断制工具类 App |
| **优点** | 用户无心理负担，一次决定；收入可预测；用户信任度高 |
| **缺点** | 后续维护成本无持续收入；新版本可能要重新收费（用户反感）；拉新成本高 |
| **适合** | 功能稳定、不需要持续运营的产品 |
| **平台抽成** | 苹果 30%（小企业 15%）；谷歌 15-30% |

---

## 二、订阅制（SaaS 模式）

按月/年付费，持续使用。

| 项目 | 说明 |
|------|------|
| **例子** | 网易云音乐、iCloud、Notion |
| **优点** | 持续收入流；可预测营收；用户门槛低（月付几块钱）；利于持续迭代 |
| **缺点** | 用户容易觉得"一直在扣钱"；需要持续提供价值；流失率管理难 |
| **变体** | 月付 / 年付 / 学生优惠 / 家庭套餐 |
| **平台抽成** | 第一年 30%，第二年起 15%（苹果）；订阅满 1 年后降为 15%（谷歌） |

### 订阅制常见策略

- **年付打折**：月付 ¥12 vs 年付 ¥98（省 32%），锁定用户
- **免费试用**：7-14 天全功能体验，降低决策门槛
- **首月优惠**：首月 ¥1，后续恢复原价
- **学生价**：教育邮箱验证，半价

---

## 三、免费 + 内购（Freemium）

基础免费，高级功能付费解锁。

| 项目 | 说明 |
|------|------|
| **例子** | 印象笔记、WPS、大部分工具 App |
| **优点** | 用户量大（免费拉新）；转化自然；口碑传播快 |
| **缺点** | 免费用户占服务器成本；付费转化率通常 2-5%；功能分级设计难 |
| **关键** | 免费版必须够用，付费版要有明显提升 |

### 功能分级思路

- 免费：核心功能完整可用
- 付费：更多存储/协作/高级分析/去除限制
- ❌ 不要在核心功能上卡墙

---

## 四、应用内购买（IAP）

在 App 内购买虚拟商品或功能。

| 项目 | 说明 |
|------|------|
| **例子** | 游戏道具、解锁高级滤镜、买表情包 |
| **优点** | 单次决策快；适合消耗型商品；用户按需购买 |
| **缺点** | 苹果强制走 IAP（抽成 30%）；不适合国内安卓（可用自有支付） |
| **类型** | 消耗型（用完再买）/ 非消耗型（买一次永久拥有）/ 订阅型 |

---

## 五、广告变现

免费使用，靠广告收入。

| 项目 | 说明 |
|------|------|
| **例子** | 今日头条、免费游戏、工具类 App（开屏广告） |
| **优点** | 用户零成本；容易获取大量用户 |
| **缺点** | 用户体验差；收入依赖流量（日活几万才有意义）；广告主依赖 |
| **形式** | 开屏广告 / Banner / 插屏 / 激励视频（看广告解锁功能） |
| **常见做法** | 免费版有广告 → 付费去广告（很常见） |

---

## 六、付费去广告

免费使用但有广告，付费移除。

| 项目 | 说明 |
|------|------|
| **例子** | 很多工具类 App |
| **优点** | 免费用户也能用；付费动力明确（烦广告）；收入=广告+去广告 |
| **缺点** | 广告影响免费用户体验；需要足够的免费用户量 |
| **定价** | 通常 ¥6-30 一次性买断 |

---

## 七、打赏/赞助

用户自愿付费支持开发者。

| 项目 | 说明 |
|------|------|
| **例子** | 开源项目、独立开发者、微信公众号赞赏 |
| **优点** | 无强制感；建立社区认同；不走应用商店抽成 |
| **缺点** | 收入极不稳定；依赖用户情感；量级小 |
| **平台** | 爱发电、Buy Me a Coffee、微信赞赏码 |
| **适合** | 个人开发者、开源项目、内容创作者 |

---

## 八、企业/团队版

面向企业收费，个人免费。

| 项目 | 说明 |
|------|------|
| **例子** | Notion 团队版、飞书、Slack |
| **优点** | 单价高（每人每月 ¥10-50）；企业付费意愿强；合同制稳定 |
| **缺点** | 销售周期长；需要企业级功能（权限/审计/SSO）；售后重 |
| **定价模式** | 按人头/月 / 按团队/年 / 按用量 |

---

## 九、数据/API 收费

免费 App，高级数据或 API 调用收费。

| 项目 | 说明 |
|------|------|
| **例子** | 天气 API、地图 API、AI 接口 |
| **优点** | 用户感知公平（用多少付多少）；适合 B2B |
| **缺点** | 计费复杂；用户对按量收费敏感 |
| **适合** | 工具型、开发者向产品 |

---

## 十、周边/增值服务

核心产品免费，周边产品收费。

| 项目 | 说明 |
|------|------|
| **例子** | 微信表情商店、主题商店、模板市场 |
| **优点** | 核心功能免费留住用户；周边不影响使用 |
| **缺点** | 需要丰富的内容生态；制作成本 |
| **适合** | 有个性化需求的产品（笔记/待办/社交） |

---

## 十一、混合模式（推荐）

现实中大多数 App 不只用一种方式，常见组合：

| 组合 | 例子 |
|------|------|
| 免费 + 订阅 | 印象笔记、Notion |
| 免费 + 去广告买断 | 大量工具 App |
| 免费 + 内购 + 广告 | 游戏常见 |
| 免费 + 订阅 + 打赏 | 独立开发者常用 |
| 免费 + 企业版 | Slack、飞书 |

## 十二、支付渠道

| 渠道 | 适用平台 | 抽成 | 备注 |
|------|---------|------|------|
| **苹果 IAP** | iOS | 30%（小企业 15%） | 强制使用，虚拟商品必须走 IAP |
| **谷歌 Play Billing** | Android 海外 | 15-30% | 虚拟商品必须走 Billing |
| **微信支付** | 国内 Android/网页 | 0.6% | 国内安卓可用，iOS 不行 |
| **支付宝** | 国内 Android/网页 | 0.6% | 同上 |
| **Stripe** | 海外网页 | 2.9%+$0.30 | 适合 SaaS 订阅 |
| **PayPal** | 全球网页 | 3-4% | 国际通用 |
| **自有支付** | 国内安卓 | 看渠道 | 国内安卓可绕过商店抽成 |

### 国内安卓的关键

国内没有统一应用商店强制 IAP，可以在 App 内直接用微信/支付宝支付，**省掉 30% 抽成**。这也是很多国内 App 只在 iOS 上架商店、安卓走自有渠道的原因。

---

## 选择建议

| 产品类型 | 推荐收费方式 |
|----------|-------------|
| 工具 App（功能稳定） | 一次性买断 或 免费+去广告 |
| 持续运营型（有云端） | 订阅制 |
| 需要大量用户 | 免费+内购（Freemium） |
| 有企业需求 | 个人免费+团队付费 |
| 独立开发者 | 免费+打赏+订阅混合 |
| 国内市场 | 微信/支付宝直付，避开苹果抽成 |
