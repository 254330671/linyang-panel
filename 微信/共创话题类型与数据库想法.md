# 共创话题类型与数据库想法

> 来源：2026-03-28 对话
> 状态：讨论中，待确认

---

## 一、话题类型

现在设计里主要是投票制，可以更丰富：

| 类型 | 交互方式 | 例子 | 数据价值 |
|------|---------|------|---------|
| **投票题** | 选一个选项 | "18岁要不要考驾照？" | 统计比例，排名 |
| **打分题** | 1-5分打分 | "你觉得晨跑有用吗？" | 平均分，推荐权重 |
| **推荐题** | 提名+点赞 | "推荐一款好用的笔记本" | 高赞=推荐库素材 |
| **经验题** | 发文字经验 | "考驾照花了多久？" | 结构化提取→知识库 |
| **二选一** | A vs B | "自习去图书馆还是咖啡厅？" | 对比数据，精准推荐 |
| **每日一问** | 轻量互动 | "今天花了多少钱？" | 活跃度+习惯数据 |

**关键区别：**
- 投票/打分/二选一 → 轻量，用户秒参与，产出统计数据
- 推荐/经验 → 重量，用户写内容，产出高质量推荐库

## 二、话题分类

按用户生活场景分，不是按功能分：

| 分类 | 话题方向 | 对接模块 |
|------|---------|---------|
| 🛒 省钱好物 | 平替、比价、好物分享 | 记账→购物推荐 |
| 🍜 吃喝 | 美食推荐、外卖避雷 | 饮食习惯 |
| 💪 运动健身 | 运动方式、装备推荐 | 健康→运动 |
| 📚 学习考试 | 考证经验、学习方法 | 成长→技能学习 |
| 💰 财务 | 理财入门、省钱技巧 | 记账→财务启蒙 |
| 🏠 生活 | 租房、出行、居家 | 生活分区 |
| 💊 健康 | 症状经验、就医分享 | 健康分区 |
| 💼 职场 | 面试、涨薪、跳槽 | 成长→职业 |
| 🎮 休闲 | 游戏、追剧、周末去哪 | 心情→休闲 |

## 三、数据库改进想法

### 1. 话题生命周期

```
创建 → 投票中（3-7天自动结题）→ 已结题 → 进入推荐库
```
自动结题很重要，不然话题永远挂着没人管。

### 2. 推荐库加"可信度"

```
recommend_cache:
  ├── topic_id
  ├── keywords[]
  ├── top_items[]（推荐结果，带赞数）
  ├── sample_size（参与人数，越多越可信）
  └── confidence（AI评估的可信度 0-1）
```
只有 3 个人投票 vs 300 人投票，价值差很多。

### 3. 用户声望系统

```
user_contributions:
  ├── user_id
  ├── topics_created（发起话题数）
  ├── topics_participated（参与话题数）
  ├── comments_liked（被点赞数）
  └── reputation_score（声望值）
```
声望高的人投票/评论权重可以更高。

### 4. 推荐库触发时机

不是用户"添加待办"时才查推荐库，而是多个场景都触发：

- 用户看天气 → 推荐"今天穿什么"的共创结果
- 用户记账 → 推荐"同类别省钱技巧"
- 用户到18岁 → 推荐"考驾照经验"
- 用户有健康症状 → 推荐"类似情况怎么处理"

## 四、数据库选型建议

| 阶段 | 方案 | 原因 |
|------|------|------|
| 现在 | PostgreSQL + Redis | 够用，成熟 |
| 用户量大了 | PostgreSQL + Redis + Elasticsearch | 搜索推荐库需要全文检索 |
| 未来 | 加图数据库（Neo4j） | "买A4纸→办公用品→推荐墨盒"关联推荐 |

## 五、核心观点

共创的核心不是"投票"，是**让用户的生活经验沉淀成可被 AI 调用的推荐资源**。投票只是收集手段，推荐库才是资产。
