# 引用外部APP vs 自研路线图

> 来源：2026-03-28 对话
> 状态：战略讨论

---

## 一、现在引用的外部服务

| 动作 | 现在用的 | 自研难度 | 值不值得自研 |
|------|---------|---------|-------------|
| 导航 | 高德地图 | ⭐⭐⭐⭐⭐ 极难 | ❌ 地图数据是壁垒 |
| 购物 | 淘宝搜索 | ⭐⭐⭐⭐⭐ 极难 | ❌ 电商平台 |
| 拨号 | tel: 协议 | 系统自带 | ✅ 本来就是原生 |
| 邮件 | mailto: 协议 | 系统自带 | ✅ 本来就是原生 |
| 搜索 | 百度/搜索引擎 | ⭐⭐ 可以做 | 🔄 可以优化 |
| 就医 | 高德搜医院 | ⭐⭐⭐⭐ 难 | ❌ 先用着 |

## 二、真正能自研的方向

不是替代外部 APP，而是在自己的生态里做更深的事。

### 购物场景 → 不替代淘宝，做"消费决策层"

```
现在：买牛奶 → 跳转淘宝搜索
未来：买牛奶 → 先看共创区推荐 → 比价 → 再跳淘宝下单
```

- 不做电商，做消费前的决策辅助
- 共创区用户推荐品牌、历史价格对比、同类替代品
- 最终下单还是跳淘宝，但决策在你这里完成
- **价值：用户买东西前先来你这里看推荐，淘宝变成"执行工具"**

### 导航场景 → 不替代高德，做"目的地管理"

- 记住用户常去的地点
- 常去的 → 一键导航
- 新地点 → 推荐共创区用户评价
- 最终导航还是跳高德，但选目的地在你这里

### 学习场景 → 完全可以自研

- 学习计划自动生成
- 进度追踪
- 共创区经验整合
- 里程碑记录

### 健康场景 → 逐步自研

- 症状→科室智能匹配
- 就医记录管理
- 用药/复诊提醒
- 体检数据趋势

### 财务场景 → 逐步自研

- 记账 → 预算分析 → 消费习惯洞察 → 共创区省钱技巧

## 三、战略思路

```
第一阶段（现在）
  外部 APP 执行，你只负责"触发"
  → 买牛奶 → 跳淘宝

第二阶段（中期）
  你在中间加一层"决策"
  → 买牛奶 → 共创推荐+比价 → 跳淘宝下单

第三阶段（长期）
  核心场景自研闭环
  → 学技能：全程在你这里
  → 健康管理：全程在你这里
  → 购物/导航：决策在你，执行在外部

终极形态
  用户生活决策层 = 你的 APP
  外部 APP 变成"工具插件"
```

## 四、核心观点

不替代外部 APP，替代"用户做决策"的过程。
地图/电商这些基础设施别碰，但"去哪里/买什么/学什么/怎么看病"这些决策在你这里完成。

## 五、形成决策层影响力后，能否将外部工具改为自研？

> 追问：已经形成决策层影响力，将外部的工具插件改为自研，有可能吗？

### 结论：部分可能，但要分场景

| 场景 | 能否自研 | 原因 |
|------|---------|------|
| **搜索** | ✅ 最容易 | 有用户数据后可做垂直搜索，不需要跟百度竞争，只搜自己生态里的内容 |
| **本地生活** | ✅ 可能 | 共创区已有用户评价数据，加上商户入驻，可替代大众点评类功能 |
| **健康管理** | ✅ 完全可以 | 不需要外部依赖，用户数据+AI分析全在自己手里 |
| **学习/技能** | ✅ 完全可以 | 纯自研闭环，不需要外部服务 |
| **记账/财务** | ✅ 完全可以 | 纯自研闭环 |
| **即时通讯** | 🔄 有难度 | 微信的壁垒是社交关系链，做不了。但 APP 内的协作聊天可以做 |
| **电商** | ⚠️ 极难 | 需要商家入驻+物流+支付+售后，重资产。但可以做"精选商城"轻模式 |
| **地图** | ❌ 几乎不可能 | 地图数据采集成本极高（高德/百度投了上百亿），用他们的 API 是唯一选择 |
| **出行/打车** | ❌ 不现实 | 需要车辆资源+调度系统+资质，重运营 |

### 可行的替代路径

#### 搜索 → 自研垂直搜索

```
现在：用户搜"怎么做红烧肉" → 跳百度
未来：用户搜"怎么做红烧肉" → 搜共创区+推荐库+技能库
```

- 不需要跟百度竞争通用搜索
- 只搜自己生态里的内容
- 共创区+推荐库+技能库=你的"知识图谱"
- 搜的结果比百度更精准（因为是真人经验）

#### 本地生活 → 共创区升级

```
现在：找附近好吃的 → 跳大众点评
未来：找附近好吃的 → 共创区用户真实评价 + AI 排序
```

- 共创区本身就是评价系统
- 比大众点评更真实（没有商家刷单）
- 可以逐步引入商户合作

#### 精选商城 → 轻模式电商

```
现在：买牛奶 → 跳淘宝
未来：买牛奶 → 你这里直接买（精选SKU，不做全品类）
```

- 不做全品类电商
- 只做"共创区高赞商品"的精选商城
- 物流用第三方（顺丰/京东物流开放平台）
- 本质是"决策+下单"一体化
- 类似"什么值得买"的模式，但更轻

#### 出行 → 做聚合不自建

```
现在：打车 → 跳滴滴
未来：打车 → 你这里叫车（聚合滴滴+高德+曹操+T3）
```

- 不自建车队
- 做出行聚合平台（API 调各打车平台）
- 用户在一个入口叫所有平台的车
- 类似高德打车的模式

### 时间线预估

| 阶段 | 时间 | 做什么 |
|------|------|--------|
| **1-2年** | 先做决策层 | 共创区+推荐库+技能系统 |
| **2-3年** | 替代轻量工具 | 自研搜索、精选商城 |
| **3-5年** | 替代中等工具 | 本地生活、出行聚合 |
| **5年+** | 深度垂直 | 健康管理全流程、学习平台 |
| **永远不碰** | — | 地图数据、全品类电商、社交关系链 |

### 成功案例参考

| 公司 | 起点 | 后来做的 |
|------|------|---------|
| **微信** | 聊天工具 | 支付、小程序、电商、出行（全生态） |
| **支付宝** | 支付工具 | 生活服务、出行、医疗、政务（全生态） |
| **美团** | 团购 | 外卖、酒旅、出行、买菜、支付（全生态） |
| **字节** | 短视频 | 电商、搜索、教育、游戏（全生态） |

**共同点：** 都是从一个切入点获得用户 → 积累数据和影响力 → 逐步扩展到相邻领域。

### 核心结论

**能，但不是一步到位。**

1. 先做好决策层（共创+推荐+AI），让用户做决定前先来你这里
2. 用户习惯形成后，逐步把"执行"也收回来
3. 能做的就自研（搜索/精选/健康/学习）
4. 做不了的就聚合（出行/外卖）
5. 永远不碰的就用 API（地图）
6. **关键：用户来了就走不掉，因为你的推荐库越来越准，别处没有**
