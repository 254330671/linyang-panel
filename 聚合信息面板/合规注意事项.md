# 合规与法律注意事项

## 当前状态（纯前端 + localStorage）
- 数据只在用户设备上，没有传输到服务器
- 不存在违法问题，最安全的状态

## 以后搭后端时的红线

### 必须做
1. **隐私政策页面** — 告知用户收集什么、怎么用、存哪
2. **用户注册时勾选同意隐私政策**
3. **敏感数据单独授权** — 健康数据（症状、BMI）属于敏感信息，《个保法》要求单独同意弹窗
4. **第三方 API 告知** — 用户数据传给 LLM API 时要告知，不能传给用户没授权的第三方
5. **UGC 内容审核** — 共创区需要举报机制和基本审核能力，否则有连带责任
6. **ICP 备案** — 上线国内服务器必须备案

### 数据库设计注意
- 把"敏感字段"标出来（健康数据、心情等）
- 后续做权限和加密时省事

### 相关法律
- 《个人信息保护法》（PIPL）
- 《网络安全法》
- 《数据安全法》

## 数据变现
- 直接卖原始数据：违法且损害用户信任，不做
- 匿名化聚合洞察：以后用户量大了可以考虑（如"24-28岁一线城市月均支出"）
- 现阶段数据用来让产品更好用，不变现
