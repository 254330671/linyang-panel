# 智能待办系统 — 产品设计文档

> 整理时间：2026-03-27 20:30
> 状态：设计阶段，待开发

---

## 一、核心定位

用户输入一句话 → AI 解析 → 生成结构化待办 + 动作链接 + 提醒

**产品原则：**
- AI 解析和建议，用户确认，不抢决定权
- 功能可以多，默认面要极简（渐进式展开）
- 主动推送有用信息，但每种推送可独立开关

## 三、AI 解析（核心功能）

用户输入一句话，AI 自动提取结构化数据：

```json
{
  "raw": "明天下午3点去市里开会",
  "time": "2026-03-28 15:00",
  "location": "市政府",
  "type": "会议",
  "actions": ["导航", "提醒"],
  "remind_before": 45
}
```

### 3.1 支持的类型及动作链接

| 类型 | 示例 | 动作链接 |
|------|------|----------|
| 📍 导航类 | 开会、办事、银行、医院、取快递 | 高德地图导航 URI |
| 🛒 购物类 | 买办公用品、买A4纸 | 淘宝/京东搜索链接 |
| 📞 通讯类 | 回客户电话、回复邮件 | tel: 拨号 / mailto: 邮箱 |
| 💰 财务类 | 还信用卡、转账 | 支付宝/银行链接 |
| 🎬 休闲类 | 看电影、约饭 | 猫眼/大众点评链接 |
| 📚 学习类 | 看书、学课程 | 豆瓣搜索/学习追踪 |
| 🏥 健康类 | 看病、买药 | 挂号平台/美团买药 |
| 🏠 生活类 | 交房租、倒垃圾 | 提醒（无外部链接） |

### 3.2 动作链接实现

不需要申请 Key，用公共 URI：
- 高德导航：`https://uri.amap.com/navigation?to=lng,lat,name&mode=car`
- 淘宝搜索：`https://s.taobao.com/search?q=关键词`
- 京东搜索：`https://so.m.jd.com/ware/search.action?keyword=关键词`
- 拨号：`tel:138xxxx`
- 邮箱：`mailto:xxx@xx.com`

手机点开自动唤起 App，没装就打开网页版。

### 3.3 智能追问（仅对有截止日逻辑的类型）

用户输入"还信用卡" → AI 检测到无时间 → 问一句：
> "信用卡还款一般有截止日，要设个提醒吗？[是] [否]"

不对所有类型都问，只对有明确截止日的类型触发。

### 3.4 关联拆分

用户输入："下周三出差去北京，见客户张总，顺便买伴手礼"

AI 拆出多个待办：
```
✅ 已创建：
  📍 下周三 北京出差 → 🗺️ 导航
  👤 见客户张总 → 📞 拨号
  🛒 买伴手礼 → 🛒 淘宝搜"伴手礼"
```
用户点"全部确认"或逐个删除。

## 四、排序规则

- 有时间 → 按紧急程度排（离现在最近的在上面）
- 没时间 → 放在有时间的下面，按添加顺序排

## 五、提醒规则

| 情况 | 规则 |
|------|------|
| 有时间的待办 | AI 根据类型自动定提前量（会议45分钟、取快递15分钟等） |
| 没时间的待办 | 不单独提醒 |
| 每日汇总 | 22:30（睡前30分钟）统一弹一次所有未完成的无时间待办 |

### 5.1 22:30 汇总提醒

不只是清单，AI 帮归类总结：
```
📋 今日待办清理

🛒 还没买：办公用品、A4纸
📞 还没打：回客户王总电话
📖 还没看：《被讨厌的勇气》第3章

💡 建议：购物可以明天下单，电话建议明天上午打
```

带 **[推迟到明天]** 按钮 — 用户点一下 → 自动创建新待办 + 原待办标完成。

## 六、时间管理（主动推送 + 用户可控）

**默认开启主动推送，用户可在设置里关掉。**

### 推送开关设计

设置页新增"智能提醒"区域：
```
智能提醒
├── 📋 每日汇总提醒     [开/关]  → 22:30 推送无时间待办
├── ⏰ 定时提醒         [开/关]  → 有时间待办到点提醒
├── 💡 空闲建议         [开/关]  → 检测到空闲时段主动建议
├── ⚠️ 冲突预警         [开/关]  → 添加待办时检测冲突
└── 📊 周/月回顾        [开/关]  → 周日/月末主动推送回顾
```
每个推送类型可独立开关，不搞一刀切。

### 推送规则

| 类型 | 触发时机 | 默认状态 |
|------|----------|----------|
| 定时提醒 | 有时间待办到提前量时 | ✅ 开 |
| 每日汇总 | 每天 22:30 | ✅ 开 |
| 空闲建议 | 检测到连续1小时以上空闲 | ✅ 开 |
| 冲突预警 | 添加待办时发现时间冲突 | ✅ 开 |
| 周回顾 | 每周日 20:00 | ❌ 关 |
| 月回顾 | 每月最后一天 | ❌ 关 |

### 6.1 查空闲
> 用户："我今天什么时候有空？"
> AI："今天下午3-4点、晚上6-8点有空。"

### 6.2 自动找空位
> 用户添加："写方案，大概2小时"
> AI："你今天6-8点有连续空闲，放这个时间段？"

### 6.3 冲突检测
> 用户添加："明天下午2点见李总"
> AI："⚠️ 明天下午2点已有'部门周会'，要改时间吗？"

### 6.4 空闲利用建议
> 用户："今晚有空做什么？"
> AI："你有3个没标时间的待办，推荐先做整理周报。"

### 6.5 工作量感知
> 用户："我这周忙吗？"
> AI："这周有23个待办，已完成16个。还有7个没完成，要排优先级吗？"

### 6.6 习惯联动
> 用户："今天运动了吗？"
> AI："还没。你7-8点正好有空，可以运动30分钟。"

### 6.7 回顾模式
> 用户："这个月怎么样？"
> AI："完成42个待办，最常做的是会议和购物。还有7个没完成，要清理吗？"

## 十一、生活分区设计

### 11.1 天气

**自动定位 + 多城市支持**
- 系统自动获取当前位置（浏览器 Geolocation API），作为默认天气源
- 用户可手动添加家人城市，带昵称和关系（🏠 我·上海、👨 爸妈·北京、💑 对象·深圳）
- 关系标签：我/爸妈/对象/朋友/其他
- 横向卡片滑动展示，点开展开详情
- 最多5个城市，第一个固定为当前定位（可手动覆盖）
- Open-Meteo API 免费，多城市就是多请求
- 定位失败时 fallback 到用户档案城市或上海

**智能穿衣建议**
- 综合温度+风速+湿度+UV+降雨+AQI
- 写成人话："今天15°C但风大体感9°，穿厚外套。下午可能阵雨，带伞。"

**家人天气关怀**
- 每次天气刷新后，检查所有家人的天气异常
- 触发条件：
  - 🌧️ 降雨概率 > 50% → "家人那边要下雨了，提醒带伞？"
  - 🥶 当日温差 > 8°C 或最低温 < 5°C → "家人那边降温了，提醒加衣服？"
  - 🌡️ 最高温 > 35°C → "家人那边高温，提醒少出门多喝水"
  - 💨 极端天气（雷暴/大雪/大雾）→ "家人那边有极端天气，关心一下"
- 推荐卡片展示在天气模块底部，轻量不打扰
- 点击可一键拨打电话/发消息（调用 tel: / sms: 协议）

**出行关联**
- 检测用户是否在家人城市附近（同一城市或临近城市）
- 节假日（春节/中秋/国庆/元旦/清明/端午）前3天 → "X节快到了，要不要回家看看？"
- 用户切换到家人城市时 → 自动显示"见一面"推荐

**季节/生活联动**
- 当季食材推荐（已实现）
- 节日关怀 → "母亲节快到了，给妈妈寄点东西？"
- 流感季（11-3月）+ 温差大 → "最近流感多发，提醒家人注意保暖"

**经期关联（用户开启才显示）**
- 健康分区加"生理"子Tab，记录经期数据
- 穿衣建议：经期前后自动加一档保暖
- 饮食推荐：补充红枣、姜茶等暖性食物
- 运动建议：经期推荐轻度运动
- 心情关联：情绪波动标注"可能和生理周期有关"
- 数据纯本地，隐私优先

### 11.2 心情

**推荐机制**
- 每次推荐从池中随机抽取，不重复最近用过的
- 一次只给一个推荐，附 [试试] [换一个] 按钮
- 结合上下文：时间、天气、待办数量、经期状态

**心情日记**
- 选完心情后可选填一句话记录原因
- 不强制，不写也完全不影响

**心情与待办联动**
- 心情差+有待办 → AI 先分析待办内容再给建议
- 不说"明天再说"，而是判断哪些必须今天做
- 推荐先做最快最简单的，建立信心
- 示例："5个待办里王总邮件今天要回，先做这个，15分钟搞定。"

**趋势洞察（用户问才答）**
> "你周三和周末心情普遍较好，周一和加班日容易低落。"

### 11.3 习惯

**固定习惯** — 用户自己加的，每天重复打勾

**今日建议** — 天气/待办/时间触发的动态建议（晴天→涂防晒，高温→多喝水）

**AI 教练 — 周末统一反馈，不每天推**

做到了 → 正反馈：
> "你这周晴天都涂了防晒，连续4天 ☀️ 再坚持3天就稳定了！"

没做到 → 换个方式试试（不说"更好的习惯"）：
> "这周有3天晴天没涂防晒，换个方式试试？
>  💡 共创区「夏日防晒心得」里有人推荐喷雾型：
>  张三：'喷一下就行，比涂的快多了' 👍128
>  👉 查看讨论"

发现模式 → 帮用户优化：
> "你连续两周周一都没运动，周一比较忙的话，改成散步？"

**共创联动规则**
- 从已结题话题库拉取，不碰当期话题
- 最多拉2条点赞最高的评论
- 格式统一：话题标题 + 评论内容 + 点赞数 + 查看链接

**建议转固定** — 用户觉得某个今日建议应该常驻，点一下加到固定列表

- 🔥 连续天数：核心动力，显示历史最长记录
- 📅 习惯日历：月视图，绿色格子=完成
- 🏷 分类标签：健康/生活/学习

### 11.4 记账

**基础功能**
- 🏷 支出分类：餐饮/交通/购物/娱乐/医疗/教育/居住/其他
- 🏷 收入分类：工资/副业/理财/红包/其他
- 📊 分类饼图：本月钱花哪了
- 💵 预算功能：设月预算，花到80%提醒
- 📈 月度对比：这个月 vs 上个月

**AI 聊天分析（用户问才答）**

每种类型都有对应分析：
```
用户："最近花销怎么变多了？"
AI："这个月餐饮1800，比上月多400，主要是外卖多了5次。
     那几天待办超过8个，忙得没时间做饭。
     💡 共创区「省钱小妙招」有人推荐周末做半成品冻起来 👍256
     你周六只有2个待办，要试试吗？"
```

**各类型推荐逻辑**

| 类型 | AI 分析什么 | 共创联动 |
|------|------------|----------|
| 餐饮超标 | 外卖频率+待办忙碌度关联 | 省钱做饭/快手菜话题 |
| 购物超标 | 频次+品类分析 | 理性消费/好物推荐话题 |
| 交通超标 | 打车频率+通勤距离 | 拼车/骑行/地铁优化话题 |
| 娱乐超标 | 消费频次 | 平替推荐/免费活动话题 |
| 收入单一 | 只有工资无副业 | 副业经验/理财入门话题 |
| 医疗支出突增 | 是否和症状记录关联 | 健康管理话题 |

**收入端也能分析**
> "这个月副业收入增加了2000，你待办里副业相关任务完成率很高，继续保持。"

**跨模块关联**
> 记账 + 待办："外卖花多了因为忙"
> 记账 + 症状："医疗支出增加，你上个月记录了3次头疼"
> 记账 + 习惯："运动花费0元，跑步是最省钱的锻炼方式"

### 12.2 身体变化里程碑

根据用户年龄+性别，到阶段自动推送一条温和的关怀提醒，附带行动建议。

**触发逻辑：**
- 用户填了年龄+性别 → 自动判断当前阶段
- 只显示当前阶段的内容，过了就不显示
- 每个阶段只推一次，用户确认后消失
- 不强制，可以跳过或关闭

**里程碑内容（男）：**

| 阶段 | 身体变化 | 行动推荐 | 联动 |
|------|---------|---------|------|
| 11-13 | 可能开始遗精 | 正常现象，不用紧张，内裤备多几条 | 🛒 购物链接 |

### 12.6 成长相册

**纯数据记录（网页版免费）：**
- 用户定期记录身高/体重/围度数据
- 自动生成生长曲线，跟同龄人标准对比
- 不评判高低，只说"你在自己的节奏里"
- 到了某个节点自动提醒量身高（联动习惯模块）
- 数据存 localStorage，加密存储

**照片记录（App 付费功能）：**

```
用户拍照/选图
  ↓
压缩（canvas 限制宽高，最长边 800px）
  ↓
转换为 ArrayBuffer
  ↓
AES-256-GCM 加密
  ↓
存入 IndexedDB（加密后的 Blob）
  ↓
云端同步（App 付费，端到端加密）
```

| 方案 | 存储方式 | 容量 | 加密 | 同步 |
|------|---------|------|------|------|
| 网页版 | 无照片功能 | — | — | — |
| App 本地 | IndexedDB | 约磁盘50% | ✅ AES-256-GCM | ❌ |
| App 云端 | 后端+CDN | 按套餐 | ✅ 端到端加密 | ✅ 自动 |

**照片功能细节：**
- 每张照片压缩后约 50-150KB（800px 最长边）
- 加解密耗时 <500ms
- 支持备注（"今天量了一下"）
- 支持身高/体重关联记录
- 支持一键导出（JSON + 图片包）
- 接近容量限制时提醒清理

**为什么照片适合做 App 付费：**
- IndexedDB 在网页版清除浏览器数据就没了，用户不放心
- App 有系统级存储保护 + 云端同步，照片才有安全感
- 照片是高情感价值功能，付费意愿强
- 免费版的数据记录已经够用，照片是锦上添花

### 13.2 急救卡（核心安全设计）

**设计原则：**
- 零泄露：锁屏只有图标，不显示任何文字信息
- 验证即全部：生物识别通过 = 看全部允许的内容
- 失败即终止：没有降级，没有兜底
- 用户主权：开不开、展示什么，全由用户决定
- 不管系统功能：系统紧急信息是用户自己的事，App 不干预

**三层解锁架构：**

```
🔓 第一层：锁屏 🆘 图标
   任何人可点击，不显示任何信息
   ↓
🔐 第二层：生物识别
   用机主的指纹/面容解锁
   ↓
📋 第三层：用户控制的展示范围
   只显示用户勾选允许的内容
   ↓
🔒 自动锁定
   5分钟或关闭 App
```

**用户设置界面：**


#### 60秒监控窗口

跌倒后持续监测心率 + 活动恢复情况：

| 时间 | 监测 | 响应 |
|------|------|------|
| T=0秒 | 跌倒触发 | 手表震动+弹窗："检测到跌倒，没事吧？" |

| T=60秒 | 用户仍无响应 | 按授权级别执行 |

## 十四、数据加密与安全

### 14.1 加密方案

使用 Web Crypto API（浏览器原生），无需额外库。

**加密算法：** AES-256-GCM
**密钥派生：** PBKDF2 + 随机 salt

### 14.2 需要加密的内容

| 内容 | 加密 | 原因 |
|------|------|------|
| 身体变化记录 | ✅ | 敏感个人健康数据 |
| 社交里程碑记录 | ✅ | 涉及隐私经历 |
| 心情日记 | ✅ | 个人情感隐私 |
| 写给未来的信 | ✅ | 高度私密 |
| 经期数据 | ✅ | 敏感健康数据 |
| 症状记录 | ✅ | 健康隐私 |
| 个人档案（年龄、性别） | ✅ | 个人身份信息 |
| 天气城市列表 | ❌ | 不敏感 |
| 考试时间库 | ❌ | 公开知识 |
| 职业路径 | ❌ | 公开知识 |
| 共创区内容 | ❌ | 社区公开内容 |

### 14.3 加密流程

```
用户设置密码（首次使用时）
  ↓
生成随机 salt（16字节）
  ↓
PBKDF2(password, salt, 100000次) → AES-256 密钥
  ↓
密钥存内存（session期间），salt + 加密数据存 localStorage
  ↓
每次读写：内存密钥 → 加解密 → 使用
  ↓
页面关闭 → 密钥销毁 → 下次需重新输入密码
```

### 14.4 用户 API 调用本地数据

```
用户问："我最近身体怎么样？"
  ↓
App 解密本地数据
  ↓
提取相关片段（最近30天症状+体重+经期）
  ↓
调用用户配置的 API（OpenAI 兼容格式）
  ↓
返回分析结果
  ↓
数据只在内存中，分析完即丢，不存任何服务器
```

**关键原则：**
- 数据只在内存中解密，不落盘到任何服务器
- 每次只传相关片段，不传全量数据
- 用户可控制哪些数据参与 API 分析

### 14.5 未来数据库对接

设计数据层抽象接口，为以后加后端做准备：

```js
// 统一接口
Storage.save(key, data)    // 自动加密
Storage.load(key)          // 自动解密
Storage.sync()             // 未来：同步到云端

// 三种实现
├── LocalEncrypted（当前）— 加密存 localStorage
├── RemoteAPI（未来）— 调后端 API，仍端到端加密
└── Hybrid（混合）— 敏感数据本地，共享数据云端
```

以后加后端只需要换实现，前端不用改。

### 14.6 图片加密存储（IndexedDB）

localStorage 存不了图片（5MB 上限），照片用 IndexedDB。

**存储方式：**
- 网页版：IndexedDB + 加密（清除浏览器数据会丢失，提醒用户定期导出）
- App：系统级存储保护 + 云端同步（端到端加密）

**IndexedDB 结构：**
```js
// Store: growth_photos
{
  id: "photo_2026_03_28",
  timestamp: 1711603200000,
  height: 165.2,
  weight: 58.0,
  encryptedBlob: ArrayBuffer,  // AES-256-GCM 加密后的图片
  note: "今天量了一下"
}
```

**加密流程：**
```
用户选图
  ↓
Canvas 压缩（最长边 800px，质量 0.8）
  ↓
转 ArrayBuffer
  ↓
AES-256-GCM 加密（复用同一密钥派生方案）
  ↓
IndexedDB 存储加密 Blob
```

**Storage 接口扩展：**
```js
Storage.saveImage(key, blob)     // 压缩 + 加密 + 存 IndexedDB
Storage.loadImage(key)           // 读取 + 解密 + 返回 Blob URL
Storage.exportAll()              // 导出全部数据（JSON + 图片）
Storage.getUsage()               // 返回已用空间，接近限制时提醒
```

### 15.1 各模块 AI 能力

| 记账 | 自动分类、月度对比、消费模式分析 |

| 健康 | 关联天气+心情+经期+症状给出综合建议 |

### 15.2 共创话题 × 待办联动

**话题使用规则**

| 场景 | 用哪期话题 |
|------|-----------|
| AI 推荐引用（待办关联） | **已结题的往期话题**，结果已定，稳定可靠 |
| 用户主动问"最近有什么热点" | **当期话题**，实时数据 |

当期话题投票中，数据不稳定，不适合做推荐引用。结题后结果固定，才是可靠知识库。

**数据流**
```
当期话题 → 投票中 → 不进推荐库
           ↓ 周五结题，发布结果
           ↓ 写入推荐库（已结题话题库）
           ↓ AI 推荐时从这里查
```

**推荐库规则**
- 推荐库 = 已结题话题，本地缓存一份
- 打开共创区时刷新缓存
- 当期话题需实时数据，离线不显示
- 库的结构、更新频率、接口格式由后端建库决定
- 前端负责对接、优化查询速度和匹配准确度

**用户添加待办时，AI 自动查推荐库匹配话题：**
```
待办：买A4纸
→ AI 查推荐库 → 找到"办公好物推荐"（已结题）
→ 推荐："共创区有人推荐：XX品牌70g性价比高，京东次日达"
→ [查看完整讨论] 跳转共创区话题页
```

### 15.3 API 调用优化

| 策略 | 做法 |
|------|------|
| 关键词预提取 | 话题加载时提取关键词缓存，本地匹配优先 |
| 解析结果缓存 | 同一待办只解析一次，存结果 |
| 精简 context | 每次只传相关数据，不传全量 |
| 离线降级 | 没网时纯本地展示，有网再补 |
| 一次调用多任务 | 待办解析+共创匹配+动作链接生成，一次 API 搞定 |

- 每天约10-20次调用
- 费用：DeepSeek ¥0.03-0.1/天，GPT-4o-mini ¥0.3-1.2/月
- 成本可忽略，放心调用

## 二十、用户档案增强 + 家庭密钥（2026-03-30 确定）

### 20.1 问题背景

各模块（天气/待办/习惯/记账/健康/共创）都需要用户档案来做个性化推荐，但当前档案信息太少且可随意改动，导致推荐质量低、数据不可信。

### 20.2 分层档案设计

#### 第一层：基础信息（注册时填写，有锁定规则）

| 字段 | 说明 | 改动规则 |
|------|------|---------|
| **年龄段** | 不填精确生日，选档：13-15 / 16-18 / 19-22 / 23-25 / 26-30 / 30+ | 每6个月可改1次 |
| **性别** | 男 / 女 / 其他 / 不愿说 | 不可改（绑定家庭密钥时由家长设定） |
| **所在城市** | 影响天气、生活推荐 | 每月可改1次 |
| **职业方向** | 选档（见下方分类），不验证 | 每月可改1次 |

职业方向选档（参考《中华人民共和国职业分类大典（2022版）》8大类 + 微信职业目录风格）：

> 标签驱动待办推荐、记账建议、习惯养成、共创区话题排序。

#### 第三层：动态画像（系统自动生成，用户不可直接编辑）

- 活跃时间段 → 调整提醒推送时间
- 待办完成率 → 调整任务拆分粒度
- 记账频率 → 判断是否需要记账提醒
- 心情记录趋势 → 调整 AI 聊天语气

> 用户可查看画像内容，可"重置画像"，但不直接编辑。

### 20.4 家庭密钥（Family Key）

#### 设计理念

低龄用户需要家长监护：锁定基础信息防篡改 + 月度成长摘要抄送家长。

#### 核心功能

**① 锁定基础信息**

- 年龄段、性别由家长设定，孩子不可改
- 孩子端看到灰色不可编辑状态
- 城市可以让孩子自己改（不影响隐私控制）

**② 月度成长摘要（每月1号自动推送）**

不是冷冰冰的数据监控，而是 **AI 帮家长读懂孩子的需求**。

示例内容：

```
📬 孩子成长摘要（2026年3月）

🎯 生活需求提醒
- 孩子13岁了，这个阶段男生可能开始长胡子了，可以聊聊要不要准备剃须刀
- 建议关注青春期卫生习惯引导

📋 待办情况
- 本月待办完成率 72%，比上月提升 5%
- 周末完成率偏低（45%），工作日较稳定
- 有3个待办连续推迟了5天以上：英语单词背诵、房间整理、跑步打卡

💡 AI观察
- 孩子最近在共创区频繁讨论编程话题，可能对计算机有兴趣
- 记账显示本月买书支出增加，学习状态不错
- 3月中旬心情连续偏低几天，建议家长多关注沟通
```

**隐私红线：**
- ❌ 不截取心情日记原文
- ❌ 不展示具体记账明细
- ✅ 只给趋势 + AI 观察 + 行动建议
- 本质：家长看到的是 AI 解读后的"育儿建议"，不是孩子的原始数据

#### 成长需求推荐库（静态表）

月报中的需求建议根据年龄+性别从预设推荐库中匹配：

| 年龄段 | 男生 | 女生 |
|-------|------|------|
| 10-12 | 注意力训练建议 | 青春期卫生引导 |
| 13-15 | 剃须/变声/长高 | 经期准备/内衣更换/护肤 |
| 16-18 | 驾照/独立出行 | 化妆/社交安全 |

AI 根据推荐库 + 孩子实际数据，生成个性化建议。

#### 关于性别绑定

**不做硬性限制**（不要求爸爸只能绑男孩、妈妈只能绑女孩）。

理由：单亲家庭、双胞胎等场景无法覆盖。

实际效果：月报推荐根据孩子年龄+性别生成，跟家长性别无关。家长性别只影响"发给谁"。

#### 密钥使用流程

```
妈妈注册 → 进入"家庭管理" → 设置孩子年龄段+性别 → 生成6位密钥
→ 发给孩子 → 孩子输入密钥 → 绑定 → 孩子端基础信息锁定
```

#### 密钥管理

| 项目 | 说明 |
|------|------|
| 密钥格式 | 6位字母数字（如 A3K9X2） |
| 绑定方式 | 一次性绑定，绑定后密钥失效 |
| 默认有效期 | 6个月 |
| 到期提醒 | 到期前7天推送给家长 |
| 续期 | 家长点一下续6个月 |
| 吊销 | 家长可随时吊销（孩子端变受限模式） |
| 多孩子 | 一个家长可生成多个密钥，每个孩子独立设置 |

#### 密钥设置项

```
家长设置：
├── 孩子年龄段（锁定，孩子不可改）
├── 孩子性别（锁定，孩子不可改）
├── 月报接收人（当前账号，默认就是自己）
├── 有效期（默认6个月）
└── 完成
```

#### 月报生成逻辑

```
每月1号触发：
1. 汇总上月各模块数据（待办/记账/心情/习惯/健康/共创）
2. 提取关键趋势（完成率变化、心情波动、消费趋势）
3. 根据年龄+性别，查"成长需求推荐库"
4. AI 生成自然语言摘要
5. 推送给家长
```
