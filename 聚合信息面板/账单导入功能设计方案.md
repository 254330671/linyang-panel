## 一、各平台账单导出格式

### 1. 微信账单

**导出路径**：我 → 服务 → 钱包 → 账单 → 常见问题 → 下载账单 → 用于个人对账

**文件格式**：CSV（UTF-8 with BOM）

**字段结构**：
```
交易时间,交易类型,交易对方,商品,收/支,金额(元),支付方式,当前状态,交易单号,商户单号,备注
2026-03-28 12:30:15,商户消费,美团外卖,午餐,-¥25.50,零钱,支付成功,xxx,xxx,
```

**特点**：
- 收/支：`-¥`表示支出，`+¥`表示收入，空表示其他
- 金额含`¥`符号
- 有表头说明行（前几行是说明，不是数据）

---

### 2. 支付宝账单

**导出路径**：我的 → 账单 → ... → 开具交易流水证明 → 用于个人对账

**文件格式**：CSV（GBK编码）

**字段结构**：
```
交易创建时间,付款时间,最近修改时间,交易来源地,类型,交易对方,商品名称,金额（元）,收/支,交易状态,服务费,成功退款,备注,资金状态
2026-03-28 12:30:15,2026-03-28 12:30:18,2026-03-28 12:30:18,其他,美团,午餐,25.50,支出,交易成功,0.00,0.00,,已清算
```

**特点**：
- GBK编码（不是UTF-8）
- 收/支：直接写"收入"/"支出"
- 金额纯数字
- 有表头说明行

---

### 3. 美团账单

**导出路径**：我的 → 我的钱包 → 账单 → 导出账单

**文件格式**：CSV

**字段结构**：
```
交易时间,交易类型,交易说明,金额,支付方式,交易状态
2026-03-28 12:30:15,外卖,麦当劳午餐,-25.50,微信支付,成功
```

**特点**：
- 格式相对简单
- 金额负数表示支出
- 没有交易单号

---

### 4. 京东账单

**导出路径**：我的 → 我的钱包 → 账单 → 导出

**文件格式**：CSV

**字段结构**：
```
交易时间,交易类型,交易说明,金额,收/支,支付方式,状态
2026-03-28 12:30:15,消费,商品购买,-99.00,支出,京东支付,成功
```

---

### 5. 拼多多账单

**导出路径**：个人中心 → 我的订单 → 更多 → 导出

**文件格式**：CSV

**字段结构**：
```
订单编号,下单时间,商品名称,金额,订单状态
PDD20260328123456,2026-03-28 12:30:15,商品名称,99.00,已完成
```

## 二、统一导入方案

### 核心思路

```
用户导出账单 → 选择文件 → 自动识别平台 → 解析字段 → 映射到统一格式 → 导入记账
```

### 统一数据格式

```json
{
  "id": "auto_gen_uuid",
  "timestamp": 1711613415000,
  "type": "expense",
  "amount": 25.50,
  "category": "餐饮",
  "note": "美团外卖-午餐",
  "source": "wechat",
  "originalData": { ... }
}
```

### 平台识别逻辑

```kotlin
fun detectPlatform(header: String): Platform {
    return when {
        header.contains("交易单号") && header.contains("商户单号") -> Platform.WECHAT
        header.contains("交易来源地") && header.contains("资金状态") -> Platform.ALIPAY
        header.contains("交易说明") && header.contains("支付方式") -> Platform.MEITUAN
        header.contains("订单编号") && header.contains("下单时间") -> Platform.PDD
        header.contains("交易类型") && header.contains("京东") -> Platform.JD
        else -> Platform.UNKNOWN
    }
}
```

## 三、关键处理

### 1. 编码处理

```kotlin
// 微信：UTF-8 with BOM
val reader = InputStreamReader(inputStream, "UTF-8")

// 支付宝：GBK
val reader = InputStreamReader(inputStream, "GBK")

// 其他：UTF-8
val reader = InputStreamReader(inputStream, "UTF-8")
```

### 2. 金额处理

```kotlin
fun parseAmount(raw: String, platform: Platform): Pair<Double, String> {
    return when (platform) {
        Platform.WECHAT -> {
            // "-¥25.50" → (25.50, "expense")
            // "+¥100.00" → (100.00, "income")
            val clean = raw.replace("¥", "").replace(",", "")
            if (clean.startsWith("-")) Pair(-clean.substring(1).toDouble(), "expense")
            else Pair(clean.toDouble(), "income")
        }
        Platform.ALIPAY -> {
            // "25.50" + "支出" → (25.50, "expense")
            Pair(raw.toDouble(), "expense")  // 需结合收/支字段
        }
        else -> Pair(raw.replace(",", "").toDouble(), "expense")
    }
}
```

### 3. 分类映射

```kotlin
// 根据商品名称自动分类
fun autoCategory(productName: String): String {
    val rules = mapOf(
        "餐饮" to listOf("外卖", "美团", "饿了么", "餐厅", "咖啡", "奶茶", "麦当劳", "肯德基"),
        "交通" to listOf("滴滴", "地铁", "公交", "加油", "停车", "高速"),
        "购物" to listOf("淘宝", "京东", "拼多多", "超市", "便利店"),
        "娱乐" to listOf("电影", "游戏", "KTV", "爱奇艺", "腾讯视频"),
        "生活" to listOf("水电", "话费", "房租", "物业", "理发"),
        "医疗" to listOf("医院", "药房", "体检", "挂号")
    )
    
    for ((category, keywords) in rules) {
        if (keywords.any { productName.contains(it) }) return category
    }
    return "其他"
}
```

### 4. 去重处理

```kotlin
// 根据交易单号去重
fun deduplicate(records: List<BillRecord>): List<BillRecord> {
    return records.distinctBy { it.sourceId }
}
```

## 五、导入流程代码

```kotlin
class BillImporter(private val context: Context) {
    
    fun import(uri: Uri) {
        // 1. 读取文件
        val inputStream = context.contentResolver.openInputStream(uri)
        
        // 2. 检测编码
        val encoding = detectEncoding(inputStream!!)
        
        // 3. 读取内容
        val content = InputStreamReader(inputStream, encoding).readText()
        
        // 4. 识别平台
        val header = content.lines().first()
        val platform = detectPlatform(header)
        
        // 5. 选择解析器
        val parser = when (platform) {
            Platform.WECHAT -> WechatBillParser()
            Platform.ALIPAY -> AlipayBillParser()
            Platform.MEITUAN -> MeituanBillParser()
            Platform.JD -> JdBillParser()
            Platform.PDD -> PddBillParser()
            else -> throw IllegalArgumentException("不支持的账单格式")
        }
        
        // 6. 解析
        val records = parser.parse(content)
        
        // 7. 去重
        val newRecords = deduplicate(records)
        
        // 8. 自动分类
        newRecords.forEach { it.category = autoCategory(it.note) }
        
        // 9. 保存
        saveToDatabase(newRecords)
        
        // 10. 通知UI
        notifyImportComplete(newRecords.size)
    }
}
```

## 七、注意事项

1. **隐私安全**：账单数据敏感，只在本地处理，不上传服务器
2. **编码问题**：支付宝是GBK，其他基本UTF-8
3. **金额格式**：各平台不同，需要统一处理
4. **去重**：同一笔交易可能重复导入，用交易单号去重
5. **分类准确性**：自动分类可能不准，允许用户手动修正
6. **大文件处理**：一年的账单可能几千条，需要异步处理
