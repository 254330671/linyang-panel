# 共创区 AI 小编 — 功能设计

## 背景
共创区话题增多后，用户难以快速了解社区在讨论什么。AI 小编负责整理、提炼、串联，降低信息获取成本。

## 四个核心功能

### 1. 每日精华摘要
每天自动生成一段社区动态，例如：
> 📌 今日共创
> "30岁前最重要的能力" 继续升温，最新一条回复获赞 120。
> "经期能不能运动" 有位医生补充了新观点。
> 新话题："毕业第一份工作该不该追求稳定" 已有 45 票。

用户30秒了解社区热点，不用自己翻。

### 2. 话题结构化整理
话题回复多了之后，AI 自动提炼：
- 正方观点（3条）
- 反方观点（2条）
- 共识点
- 争议焦点

不是替用户判断，是帮他们看清讨论全貌。

### 3. 自动补充背景知识
新话题下方自动补充相关背景，如社保话题自动附带五险一金基础信息。
省得评论区反复解释基础事实。

### 4. 跨话题关联
发现新话题和旧话题有关联时自动提醒：
> 🔗 这个话题和上月热门"租房避坑指南"相关，有 8 条经验可以参考。

让社区知识串联而非散落。

## 成本分析
- 每日简报：输入500-1000 tokens + 输出200 tokens，每天1次
- DeepSeek：每月几毛钱，几乎忽略
- GPT-4o-mini：每天不到 $0.001，一个月几分钱
- 聊天比简报贵得多（上下文越聊越长），简报是轻量任务
- 方案选择：聊天用用户Key（费钱），简报可平台统一跑（成本低可承担）

## 开发优先级
1. **每日精华摘要** — 最简单、用户感知最强、开发量最小，先做
2. 自动补充背景知识
3. 话题结构化整理
4. 跨话题关联

## 实现方案
- 用户自带 API Key：AI 聊天用用户的 Key
- 轻量任务（简报）：可平台统一跑，用最便宜的模型
- 生成频率：每天一次，算力很省
- 触发时机：用户打开共创区时检查是否有新的摘要需要生成
