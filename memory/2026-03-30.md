## 档案大改讨论

- 用户提出"档案大改"，指用户身份档案
- 问题：各模块需要档案做个性化推荐，但当前只有ID+昵称，信息太少且可随意改动
- 解决方案：分层档案（基础信息+兴趣标签+动态画像）+ 家庭密钥
- 家庭密钥：家长锁定孩子基础信息 + 月度成长摘要抄送
- 月报理念：不是监控，是AI帮家长读懂孩子需求（趋势+建议，不暴露原始数据）
- 具体推荐：根据成长需求推荐库（年龄+性别），如剃须刀/卫生巾等
- 不做家长性别硬绑定，考虑单亲家庭等场景
- 职业方向细化：参考国家职业分类大典，15个大类+细分方向，二级选择
- 已更新设计文档第二十章 + MEMORY.md

## P3 实现 — 家庭密钥

### 新增文件
- `screens/settings/family_key_page.dart` — 家庭密钥管理页（完整19KB）

### 家庭密钥功能
- **家长端**：创建密钥 → 选择孩子年龄段+性别 → 生成6位随机码（排除易混淆字符I/O/0/1）→ 显示给家长
- **孩子端**：输入6位密钥 → 绑定 → 自动设置年龄段和性别 → 锁定（灰色+锁图标）
- **密钥管理**：查看状态（待绑定/已绑定/已过期/已吊销）/ 续期6个月 / 吊销
- **到期提醒**：到期前7天显示警告
- **多孩子**：一个家长可生成多个密钥
- **月报生成**：每月1号自动生成成长摘要（生活需求+待办率+记账+心情+AI观察）
- **月报通知**：集成到 NotificationService，每月1号推送

### 设置页改动
- 新增"家庭管理"入口卡片 → 进入 FamilyKeyPage

### NotificationService 改动
- 新增 `_checkMonthlyReport()` — 每月1号生成月报+推送
- 新增密钥到期前7天续期提醒

### ProfileService 新增
- `generateMonthlyReport()` — 月报生成逻辑
- `getExpiryWarning()` — 到期提醒检查

### 本地版本说明
- 密钥存储在同一个 profile 里（同一设备模拟），实际需要后端跨设备验证
- 月报数据从本地画像读取
