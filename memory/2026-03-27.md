## 智能待办系统设计讨论（详见 聚合信息面板/智能待办系统设计.md）
- 核心思路：用户一句话 → AI解析 → 结构化待办+动作链接
- 支持8种类型，每种配动作链接（导航/购物/拨号等）
- 动作链接用公共URI，不需要申请Key
- 排序：有时间按紧急程度，没时间放下面
- 提醒：AI自动定提前量，22:30汇总无时间待办
- 22:30汇总带AI归类总结+"推迟到明天"一键操作
- 时间管理功能：纯被动，用户问才答（不主动打扰）
- 语音输入：Web Speech API
- 版本策略：网页版免费纯本地，App付费+协作+群同步
- 多端统一后端：网页/App/语音助手都调同一套API
- 开发节奏：Phase1网页版 → Phase2后端 → Phase3 App → Phase4语音助手
